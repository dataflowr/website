Attention with RNNs

The first attention mechanism was proposed in Neural Machine Translation by Jointly Learning to Align and Translate by Dzmitry Bahdanau, Kyunghyun Cho, Yoshua Bengio (presented at ICLR 2015).

The task considered is English-to-French translation and the attention mechanism is proposed to extend a seq2seq architecture by adding a context vector cic_ici​ in the RNN decoder so that, the hidden states for the decoder are computed recursively as si=f(si−1,yi−1,ci)s_i = f(s_{i-1}, y_{i-1}, c_i)si​=f(si−1​,yi−1​,ci​) where yi−1y_{i-1}yi−1​ is the previously predicted token and predictions are made in a probabilist manner as yi∼g(yi−1,si,ci)y_i \sim g(y_{i-1},s_i,c_i)yi​∼g(yi−1​,si​,ci​) where sis_isi​ and cic_ici​ are the current hidden state and context of the decoder.

Now the main novelty is the introduction of the context cic_ici​ which is a weighted average of all the hidden states of the encoder: ci=∑j=1Tαi,jhjc_i = \sum_{j=1}^T \alpha_{i,j} h_jci​=∑j=1T​αi,j​hj​ where TTT is the length of the input sequence, h1,…,hTh_1,\dots, h_Th1​,…,hT​ are the corresponding hidden states of the decoder and ∑jαi,j=1\sum_j \alpha_{i,j}=1∑j​αi,j​=1. Hence the context allows passing direct information from the 'relevant' part of the input to the decoder. The coefficients (αi,j)j=1T(\alpha_{i,j})_{j=1}^T(αi,j​)j=1T​ are computed from the current hidden state of the decoder si−1s_{i-1}si−1​ and all the hidden states from the encoder (h1,…,hT)(h_1, \dots, h_T)(h1​,…,hT​) as explained below (taken from the original paper):

(Self-)Attention in Transformers

We now describe the attention mechanism proposed in Attention Is All You Need by Vaswani et al. First, we recall basic notions from retrieval systems: query/key/value illustrated by an example: search for videos on Youtube. In this example, the query is the text in the search bar, the key is the metadata associated with the videos which are the values. Hence a score can be computed from the query and all the keys. Finally, the matched video with the highest score is returned.

We see that we can formalize this process as follows: if QsQ_sQs​ is the current query and KtK_tKt​ and VtV_tVt​ are all the keys and values in the database, we return

Ys=∑t=1Tsoftmaxt(score(Qs,Kt))Vt, Y_s = \sum_{t=1}^T\text{softmax}_{t}(\text{score}(Q_s, K_t))V_t, Ys​=t=1∑T​softmaxt​(score(Qs​,Kt​))Vt​,

where ∑t=1Tsoftmaxt(score(Qs,Kt))=1\sum_{t=1}^T\text{softmax}_{t}(\text{score}(Q_s, K_t))=1∑t=1T​softmaxt​(score(Qs​,Kt​))=1.

Note that this formalism allows us to recover the way contexts were computed above (where the score function was called the alignment network). Now, we will change the score function and consider dot-product attention: score(Qs,Kt)=QsTKtd \text{score}(Q_s, K_t) = \frac{Q_s^TK_t}{\sqrt{d}}score(Qs​,Kt​)=d​QsT​Kt​​. Note that for this definition to make sense, both the query QsQ_sQs​ and the key KtK_tKt​ need to live in the same space and ddd is the dimension of this space.

Transformer block

To finish the description of a transformer block, we need to define two last layers: Layer Norm and Feed Forward Network.

The Layer Norm used in the transformer block is particularly simple as it acts on vectors and standardizes it as follows: for x∈Rdx\in \mathbb{R}^dx∈Rd, we define

mean(x)=1d∑i=1dxi∈Rstd(x)2=1d∑i=1d(xi−mean(x))2∈R \text{mean}(x) =\frac{1}{d}\sum_{i=1}^d x_i\in \mathbb{R}\\ \text{std}(x)^2 = \frac{1}{d}\sum_{i=1}^d(x_i-\text{mean}(x))^2\in \mathbb{R} mean(x)=d1​i=1∑d​xi​∈Rstd(x)2=d1​i=1∑d​(xi​−mean(x))2∈R

and then the Layer Norm has two parameters γ,β∈Rd\gamma, \beta\in \mathbb{R}^dγ,β∈Rd and

LN(x)=γ⋅x−mean(x)std(x)+β, LN(x) = \gamma \cdot \frac{x-\text{mean}(x)}{\text{std}(x)}+\beta, LN(x)=γ⋅std(x)x−mean(x)​+β,

where we used the natural broadcasting rule for subtracting the mean and dividing by std and ⋅\cdot⋅ is component-wise multiplication.

A Feed Forward Network is an MLP acting on vectors: for x∈Rdx\in \mathbb{R}^dx∈Rd, we define

FFN(x)=max⁡(0,xW1+b1)W2+b2, FFN(x) = \max(0,xW_1+b_1)W_2+b_2, FFN(x)=max(0,xW1​+b1​)W2​+b2​,

where W1∈Rd×hW_1\in \mathbb{R}^{d\times h}W1​∈Rd×h, b1∈Rhb_1\in \mathbb{R}^hb1​∈Rh, W2∈Rh×dW_2\in \mathbb{R}^{h\times d}W2​∈Rh×d, b2∈Rdb_2\in \mathbb{R}^db2​∈Rd.

Each of these layers is applied on each of the inputs given to the transformer block as depicted below:

Note that this block is equivariant: if we permute the inputs, then the outputs will be permuted with the same permutation. As a result, the order of the input is irrelevant to the transformer block. In particular, this order cannot be used. The important notion of positional encoding allows us to take order into account. It is a deterministic unique encoding for each time step that is added to the input tokens.
